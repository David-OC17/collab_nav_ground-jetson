Sweep Parameter Reference

Quick-reference for the 83-run overnight sweep. Groups match run_matrix.py.

Frame extraction  ·  groups D

target_fps float 3–7 How many frames per second to sample from the video. Lower = fewer frames to stitch, faster but may miss coverage. Higher = denser overlap, slower, higher risk of static-frame noise.
min_movement float 0.010–0.020 Minimum feature displacement (fraction of frame diagonal) required to keep a frame. Frames below this are treated as the drone hovering and skipped. Lower = accept nearly-static frames; higher = only keep frames with clear motion.

Feature backend  ·  groups A, H

feature_extractor categorical sift — classic hand-crafted detector, 5000 keypoints, CPU-only.
superpoint — learned neural detector, 256 keypoints (GPU), more robust on textureless floors.
feature_matcher categorical ratio_test — Lowe's ratio test + mutual cross-check. Works with both extractors.
lightglue — learned matcher (GPU). SuperPoint only. Produces fewer but more reliable matches on difficult scenes.
feature_exclude_hsv categorical none — detect features everywhere.
blue_tape — mask out the blue grid tape before feature detection so keypoints don't land on the repetitive grid (only meaningful for SIFT; SuperPoint ignores this).

Matching quality  ·  groups B, F, G

match_ratio float 0.65–0.80 Lowe's ratio threshold for descriptor matching. Lower = stricter (fewer, more reliable matches). Higher = more permissive (more matches, more ambiguous ones). Ignored by LightGlue.
mad_factor float 4–6 Median-Absolute-Deviation gate applied to match displacements before RANSAC. Rejects matches whose shift deviates from the median by more than this × MAD. Lower = tighter pre-filter, helps on repetitive grids. Too low = over-rejection.
feature_exclude_dilate_px int 3–12 How many pixels to dilate the blue-tape exclusion mask outward. Without dilation, keypoints latch onto tape edges (still grid-locked). Larger = cleaner exclusion; too large = fewer features overall.
min_inliers int 8–20 Minimum RANSAC inliers required to accept a frame placement. Lower = place more frames (including uncertain ones). Higher = only place high-confidence alignments, possibly leaving gaps.

Grid alignment  ·  groups A

use_grid_intersections bool Whether to detect blue tape grid intersections and use them as a second RANSAC stage after feature matching. The grid provides sub-pixel geometric anchors that tighten rotation and scale estimates. Disabling reverts to feature-only alignment.

Temporal coverage  ·  groups C, I

lookback int 4–8 How many recently placed frames to try matching against for each new frame. Higher = more matching attempts per frame, catches the best recent overlap, slower per frame.
keyframe_interval int 5–15 Store a long-range keyframe every N successfully placed frames. These are used for re-localisation when the drone revisits an area. Lower = denser keyframe coverage (better loop closure), more memory. Higher = fewer keyframes.

Pose graph & solver  ·  groups A, E, J

use_pose_graph bool Run a global least-squares solve over keyframe poses after the online pass. Corrects the accumulated drift visible as bowed-grid or stacked-cards artifacts. The live map is unaffected; the corrected map is produced by finalize().
use_fiducials bool Detect ArUco markers per frame and add them as zero-ambiguity loop-closure constraints in the pose graph. Each marker ID seen in multiple frames ties those frames together with high-accuracy corner correspondences. Requires use_pose_graph=True to have any effect.
pg_marker_weight float 5–25 How strongly marker-corner constraints pull the solve compared to odometry edges. Higher = markers dominate the correction; lower = feature-loop edges have more say. Subpixel corners are very accurate so higher values are usually safe.
pg_huber_delta float 2–8 Huber loss threshold (pixels) for the global solve. Residuals below this are treated as normal least-squares; above it, their influence is down-weighted. Lower = tighter rejection of bad loop closures; higher = more permissive.
